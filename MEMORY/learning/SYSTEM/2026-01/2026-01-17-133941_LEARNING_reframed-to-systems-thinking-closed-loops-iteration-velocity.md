## The Real Framework: Closed Loops & Iteration Velocity

### The Core Insight

> **If you don't have a feature in a workable LOOP, it's just overhead.**

This is the filter. Everything PAI has or builds must answer:

1. **Is this a closed loop?** (Does it complete without human intervention in the middle?)
2. **What's the minimum viable closure?** (Smallest thing that makes it not-overhead)
3. **What's the iteration frequency?** (How fast can you cycle through learn‚Üído‚Üílearn?)
4. **Is it deterministic enough to steer?** (Can you repeat it reliably to see if changes helped?)

## The Iteration Frequency Bias

### Steering > Certainty

> **Bias iteration frequency with active steering OVER iteration outcome certainty.**

Translation:
- Fast iterations with course corrections beat slow iterations with perfect outcomes
- You learn more from 10 quick cycles than 1 "perfect" attempt
- Determinism enables steering‚Äîif you can't repeat it, you can't steer it

### Determinism for Iterate-peatability

| Property | Why It Matters |
|----------|---------------|
| **Repeatable** | Same input ‚Üí same output ‚Üí you can compare runs |
| **Observable** | You can see what happened ‚Üí you can diagnose |
| **Adjustable** | One knob at a time ‚Üí you know what caused change |
| **Measurable** | Quantifiable outcome ‚Üí you know if steering helped |

1. **For each PAI subsystem:** Is this a closed loop or does it require human intervention mid-cycle?

### Iteration Velocity Questions

4. **What's the cycle time?** From "I do something" to "I see the effect" ‚Äî how long?
5. **What blocks faster iteration?** Where is latency hiding?
6. **Can I run the same thing twice and compare?** Is it deterministic enough?

### Steering Questions

7. **What knobs exist?** What can I adjust?
8. **Can I see the effect of turning a knob?** Is the feedback visible?
9. **Is the causality traceable?** When something changes, can I know why?

### Overhead Detection Questions

10. **What's running but not closing?** Features that exist but don't complete loops
11. **What requires me to remember to use it?** If I have to remember, it's not a system
12. **What would I not notice if it stopped?** That's overhead
